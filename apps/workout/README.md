# Workout Tracker

A PWA for tracking workouts, served at [workout.jordanscamp.site](https://workout.jordanscamp.site). Designed for offline-first use — log workouts without connectivity and sync when back online.

**Status**: Scaffold only. Routes and offline infrastructure are in place; UI and backend models are not yet built.

## Running locally

```bash
pnpm --filter workout dev
```

## Tech stack

- React + TypeScript + Vite
- vite-plugin-pwa — service worker generation with workbox runtime caching
- React Router — client-side routing
- Zustand — state management

## Routes

| Path | Page | Description |
|------|------|-------------|
| `/` | Dashboard | Overview / landing page |
| `/log` | Log Workout | Record a new workout session |
| `/history` | History | View past workouts |
| `/exercises` | Exercises | Browse and manage exercises |

## Offline strategy

The service worker (generated by vite-plugin-pwa) precaches the app shell and uses a NetworkFirst strategy for API calls to `api.jordanscamp.site/api/workout/*`, falling back to cached responses when offline.
